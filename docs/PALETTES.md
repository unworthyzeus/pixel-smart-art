## Palette Modes

PixelSmart Art supports three palette modes:

### 1. Predefined Palettes
Classic gaming and artistic palettes with historically accurate colors.

### 2. Extracted Palettes
AI-powered extraction of dominant colors from your source image using k-means clustering.

### 3. Custom Palettes
Create your own palette by picking colors or importing hex values.

## Classic Gaming Palettes

### Game Boy (4 colors)

The original Nintendo Game Boy (1989) used a 4-shade green LCD display.

| Color | Hex | Usage |
|-------|-----|-------|
| Dark | `#0f380f` | Shadows, outlines |
| Medium-Dark | `#306230` | Mid-tones |
| Medium-Light | `#8bac0f` | Highlights |
| Light | `#9bbc0f` | Brightest areas |

**Best For**: Nostalgic Game Boy recreations, minimalist art

---

### NES Classic (54 colors)

The Nintendo Entertainment System (1983) had a unique 54-color palette generated by its PPU (Picture Processing Unit).

The NES palette includes:
- Black and white
- 3 grays
- Full spectrum of hues in multiple luminance levels
- Some "forbidden" colors that caused visual glitches

**Best For**: 8-bit era aesthetics, detailed pixel art

---

## Color Extraction

### How It Works

The AI palette extraction uses **k-means clustering**:

1. **Sample Pixels**: Every 4th pixel is sampled for speed
2. **Initialize Centroids**: First k unique colors become starting points
3. **Iterate**: 10 iterations of:
   - Assign each pixel to nearest centroid
   - Move centroid to cluster average
4. **Sort**: Final colors sorted by luminance (dark to light)

### Tips for Best Results

- **More colors** (16-32) for detailed images
- **Fewer colors** (4-8) for stylized look
- Works best on images with **clear color separation**
- May struggle with **gradients** (use fewer colors)

## Custom Palettes

### Creating Custom Palettes

1. Click **Custom** in palette mode
2. Use color pickers to add/edit colors
3. Click **+** to add new colors
4. Click **×** on hover to remove colors

### Tips for Good Custom Palettes

1. **Include black and white** for full range
2. **Spread luminance evenly** across colors
3. **Include transition colors** between main hues
4. **Test with different images** before finalizing
5. **Limit to 4-16 colors** for true pixel art feel

### Example Custom Palette Ideas

**Minimal Dark**:
```
#0a0a0a, #1a1a2e, #16213e, #0f3460, #e94560
```

**Warm Monochrome**:
```
#1a0a00, #3d1c00, #612e00, #854000, #a95200, #cd6400
```

**Neon on Black**:
```
#000000, #ff0055, #00ff88, #00aaff, #ffff00, #ffffff
```

## Color Distance Algorithm

PixelSmart Art uses a **perceptually-weighted** color distance formula:

```javascript
function colorDistance(c1, c2) {
  const rMean = (c1.r + c2.r) / 2;
  const dR = c1.r - c2.r;
  const dG = c1.g - c2.g;
  const dB = c1.b - c2.b;
  
  return Math.sqrt(
    (2 + rMean / 256) * dR * dR +
    4 * dG * dG +
    (2 + (255 - rMean) / 256) * dB * dB
  );
}
```

This formula accounts for human color perception:
- **Green** has highest weight (most sensitive)
- **Red** weight varies based on overall red content
- **Blue** weight varies inversely with red
